# Verify API (project-owned)

Generated by `/builder-setup-harness` for this API repo. Grows via the Gotchas loop.

End-to-end verification by calling the running API and asserting the response — the counterpart to unit tests.

## How it works here

1. Bring the stack up: `RUN_LOCAL_REF` <!-- setup fills, or points at the builder-run-local skill -->
2. Base URL: `API_BASE_URL` <!-- setup fills -->
3. Auth, if any: `AUTH_SETUP` <!-- setup fills -->
4. Call the changed endpoint(s) and assert **status code and response shape**.

Example shape (setup replaces with a real call verified during setup):

```
$ curl -s -o /dev/null -w '%{http_code}' API_BASE_URL/health
200
```

## Rules

- Assert both status and body shape, not just a 200.
- Cover the error path too (bad input → expected 4xx), not only the happy path.
