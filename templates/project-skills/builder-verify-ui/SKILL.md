# Verify UI (project-owned)

Generated by `/setup-harness` for this frontend repo. Grows via the Gotchas loop.

End-to-end verification by interacting with the running app the way a user would — the counterpart to unit tests, which don't prove the feature is usable.

## How it works here

1. Bring the app up: `DEV_SERVER_COMMAND` <!-- setup fills + verified during setup -->
2. App URL: `APP_URL` <!-- setup fills -->
3. Drive the changed flow: navigate, interact, assert the resulting state.
4. Screenshot the working feature (embed in the report's proof-of-work section).
5. Check the browser console: **zero new errors or warnings.**

## Rules

- Assert observable state, not just "the page loaded".
- A screenshot with a console error in the tab is not a pass.
- Record any flakiness or manual step in Gotchas so it can be automated later.
