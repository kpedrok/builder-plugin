# Builder Prototype (project-owned)

Generated by `/setup-harness` for this frontend repo. Grows via the Gotchas loop.

Explore a UI idea as a few genuinely distinct variations on a throwaway route/page before building it for real. Pick a winner (or a blend) with the human, then hand off to the `feature` pipeline to build only that.

## How it works here

- Throwaway location: `PROTO_LOCATION` <!-- setup fills: e.g. a /playground route, a Storybook story, a scratch page -->
- Start the dev server: `DEV_SERVER_COMMAND` <!-- setup fills + must have run this during setup -->
- Reuse existing components, primitives, and design tokens per `docs/agents/design.md` — do not invent new ones for a throwaway.

## Rules

- Variations must be genuinely different, not recolors.
- Nothing here ships. Delete the throwaway route once a direction is chosen.
- The chosen direction goes through `feature` (ALIGN → PLAN → …) like any other build.
